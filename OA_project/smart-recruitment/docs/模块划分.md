# 企业智慧招聘 OA 项目模块划分

> 本文档按当前仓库源码整理，目录中标注“规划”或“待实现”的内容不是当前已经存在的代码。业务实现状态以各模块的 Controller、Entity、Mapper、配置文件和 Git 提交为准。

## 一、项目总体结构

```text
smart-recruitment/
├── recruitment-backend/       Spring Boot 后端多模块工程
├── recruitment-web/           Vue 3 + Vite 前端
├── recruitment-web-prototype/ 前端原型/备用工程
├── sql/                       MySQL 初始化脚本
├── docs/                      接口、设计、开发日志和配置文档
└── docker/                    Redis Stack、Elasticsearch、Nacos 编排
```

当前后端服务端口：核心招聘服务 `8080`，AI 服务 `8081`。前端开发服务器默认为 `3000`，通过 Vite 代理访问核心服务的 `/api`。

## 二、后端 Maven 模块

```text
recruitment-backend/
├── pom.xml
├── recruitment-common/
│   ├── common-core/
│   ├── common-web/
│   ├── common-security/
│   └── common-redis/
├── recruitment-api/
├── recruitment-service/
└── recruitment-ai-service/
```

父工程当前使用 JDK 21、Spring Boot 3.2.5、Spring Cloud 2023.0.1、Spring Cloud Alibaba 和 MyBatis-Plus。后端按业务域分包，但多数业务域目前只有 Controller、Entity 和 Mapper，Service 层仍需补齐。

## 三、公共模块

### 3.1 common-core

当前代码：

```text
com.recruitment.common.core/
├── constant/   CommonConstant
├── context/    UserContext
├── exception/  BusinessException、ErrorCode
└── model/      Result、PageResult
```

职责：

- 统一响应 `Result<T>`。
- 统一分页对象 `PageResult<T>`，字段为 `records`、`total`、`pageNum`、`pageSize`。
- 业务异常和错误码。
- 当前请求用户上下文和公共常量。

原规划中的日期工具、JSON 工具、枚举、数据脱敏工具目前尚未形成独立源码目录。

### 3.2 common-web

当前代码：

```text
com.recruitment.common.web/
├── advice/       GlobalExceptionHandler
├── annotation/   OperationLog
├── aspect/       OperationLogAspect
├── config/       WebMvcConfig
└── interceptor/  RequestIdInterceptor、UserContextInterceptor
```

职责：

- 全局异常处理。
- 操作日志注解和切面。
- Web MVC 配置。
- 请求 ID 和用户上下文拦截器。

统一参数校验、通用分页参数解析等能力目前尚未单独实现。

### 3.3 common-security

当前代码：

```text
com.recruitment.common.security/
├── config/   SecurityConfig
├── filter/   JwtAuthenticationFilter
├── jwt/      JwtUtil
└── model/    LoginUser
```

职责：

- Spring Security 基础配置。
- JWT 生成、解析和请求过滤。
- 登录用户模型。

密码加密、UserDetailsService、细粒度权限处理器和数据权限辅助类尚未形成完整实现，认证 Controller 目前也仍是接口骨架。

### 3.4 common-redis

当前代码：

```text
com.recruitment.common.redis/
├── config/  RedisConfig
└── util/    RedisUtil
```

职责：

- Redis 客户端基础配置。
- Redis 常用读写封装。

分布式锁、验证码、Token 黑名单和防重复提交目前属于规划能力，不应视为已经提供的公共 API。

## 四、服务间接口模块 recruitment-api

当前代码：

```text
com.recruitment.api/
├── client/
│   └── AiServiceClient.java
└── dto/
    ├── AiMatchRequest.java
    ├── AiMatchResponse.java
    ├── AiQuestionRequest.java
    ├── AiQuestionResponse.java
    ├── AiResumeParseRequest.java
    └── AiResumeParseResponse.java
```

`AiServiceClient` 使用 OpenFeign 调用 AI 服务，当前声明的远程接口为：

- `/api/ai/resume/parse`
- `/api/ai/match`
- `/api/ai/question/generate`

原文中的 `AiInterviewRequest`、`AiInterviewResponse` 当前不存在；模拟面试接口目前由 AI 服务 Controller 单独声明，尚未抽取到该模块的 DTO。

## 六、AI 服务 recruitment-ai-service

入口类：`com.recruitment.ai.RecruitmentAiApplication`

当前实际目录：

```text
com.recruitment.ai/
├── config/
│   └── AiModelConfig
├── evaluation/
│   └── service/EvaluationService
├── matching/
│   └── controller/MatchingController
├── mockinterview/
│   └── controller/MockInterviewController
├── question/
│   └── controller/QuestionController
├── resume/
│   └── controller/ResumeController
└── vector/
    └── service/VectorService
```

### 6.1 config

`AiModelConfig` 负责 AI 模型相关配置入口。OpenAI API Key、Base URL 和模型参数位于 `application.yml`，Redis/Nacos 联调配置位于 `application-dev.yml`。

### 6.2 resume

`ResumeController` 已定义：

- `/api/ai/resume/parse`
- `/api/ai/resume/evaluate`
- `/api/ai/resume/optimize`

当前仅有 Controller，提示词、结构化结果转换、真实模型调用和 DTO 业务编排尚未实现；解析接口使用 `recruitment-api` 中的 `AiResumeParseRequest/Response`。

### 6.3 matching

`MatchingController` 定义 `/api/ai/match`，使用 `AiMatchRequest` 和 `AiMatchResponse`。`VectorService` 位于独立 `vector.service` 包中，但当前没有匹配 Service、评分计算器或 Redis Stack 向量 Repository。

### 6.4 question

`QuestionController` 定义 `/api/ai/question/generate`，使用 `AiQuestionRequest` 和 `AiQuestionResponse`。正式面试题的提示词、题目生成业务和面试官权限校验尚未实现。

### 6.5 mockinterview

`MockInterviewController` 定义：

- `/api/ai/mock-interview/generate`
- `/api/ai/mock-interview/record`
- `/api/ai/mock-interview/follow-up`
- `/api/ai/mock-interview/report/{sessionId}`

当前只有 Controller，模拟面试会话、回答记录、追问和报告 Service 尚未实现。模拟面试接口应与面试官使用的正式面试题接口保持权限和路径隔离。

### 6.6 evaluation 与 vector

当前 `evaluation` 只有 `EvaluationService`，`vector` 只有 `VectorService`，尚未形成完整的 Controller、Repository、Model 或 Redis Stack 向量读写实现。

## 九、基础设施与配置

配置文件：

- `recruitment-service/src/main/resources/application.yml`

## 十、当前开发状态与建议顺序

当前后端属于“工程骨架 + Controller/Entity/Mapper 初始实现”阶段，主要缺口是 Service、DTO/VO 参数绑定、数据库 CRUD、认证业务、文件存储、Elasticsearch 同步和 AI 真实调用。

建议后续顺序：

1. 补齐 AuthService、用户查询、密码加密和 JWT 登录闭环。
2. 为 Job、Resume、Application、Interview 建立 DTO、Service、Mapper 查询和状态流转。
3. 完成文件上传落盘、简历解析入口和真实文件下载流。
4. 完成 Elasticsearch 索引、同步、搜索和高亮。
5. 完成 AI 服务真实模型调用、匹配评分、正式题目和模拟面试会话。
6. 完成 dashboard 聚合统计和前后端真实联调。
7. 最后再将前端 Mock 数据替换为数据库和真实服务返回。

## 十一、与原规划文档的主要修正

- 删除了当前不存在的 `service`、`dto`、`vo`、`document`、`repository` 目录的“已实现”表述。
- 将 `AiInterviewRequest`、`AiInterviewResponse` 从当前模块清单中移除，因为源码中不存在这两个 DTO。
- 将 `MockInterview` 和 `MockInterviewRecord` 按当前代码归入核心服务的 `interview.entity`，并注明其与 AI 服务的边界待明确。
- 将职位、简历、搜索和文件模块的现状标记为 Controller/Mapper 骨架，避免把规划能力误写成已完成能力。
- 补充了前端实际存在的 `file.js`、`search.js`、`stores/inbox.js` 和 `utils/websocket.js`。
