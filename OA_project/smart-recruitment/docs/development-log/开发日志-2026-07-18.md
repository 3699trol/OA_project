## 四、开发工作记录

### 4.1 简历与文件接口

- 建立 `Resume` 实体和 `ResumeMapper`，覆盖用户、文件地址、解析内容、AI 评分及自我评价等字段。
- 定义简历详情、保存、更新、AI 解析和 AI 评估接口。
- 建立文件上传和下载接口，使用 `MultipartFile` 作为上传入口。
- 在数据库中补充教育经历、工作经历及简历扩展字段。
- 当前接口路由和参数结构已建立，真实文件落盘、元数据保存和删除逻辑仍需补充。

### 4.2 API 标准化

- 统一用户、职位、投递、面试等列表接口的 `list + total` 返回结构。
- 补充职位分类、操作日志、系统配置等管理接口入口。
- 统一前端 API 模块命名和请求方式，减少页面中的硬编码 URL。
- 明确 HTTP 状态与业务状态分离，业务接口统一返回 `Result<T>`。
- 调整配置冲突，保证 `local` 与 `dev` profile 可以按场景选择。

### 4.3 AI 服务与服务发现

- 修复 AI 服务 Spring AI 相关依赖，保证应用启动类可加载。
- 为核心服务和 AI 服务配置 Nacos 服务发现参数。
- 核心服务调用 AI 服务时支持 Nacos 服务名发现和固定 URL 直连两种方式。
- 统一 AI 服务地址、Nacos 地址和模型密钥的环境变量入口。
- 保留真实模型不可用时的 Mock 降级方向。

### 4.4 前端简历与 UI

- 完成求职者简历编辑、简历评估、职位推荐和模拟面试等页面结构。

- 统一 Element Plus 组件、表单、列表、卡片和加载状态的视觉风格。
- 建立内置 Mock 请求拦截器，模拟认证、职位、投递、面试、简历和 AI 接口。
- Mock 接口保持与真实 API 相同的路径和响应结构，便于后续替换。

### 4.5 Mock 核心业务链路

- 支持四种角色的快速登录和对应页面跳转。

## 七、问题与处理记录

### 7.1 真实后端进度影响前端联调

建立与真实 API 契约一致的 Mock 拦截器，让前端先完成流程验证；真实 Service 完成后通过关闭 Mock 直接切换。

### 7.2 Nacos 与 AI 服务配置发生冲突

拆分 `local`、`dev` 配置，并允许通过 `AI_SERVICE_URL` 选择固定地址；变量为空时使用 Nacos 服务发现。

### 7.3 文件上传链路未完成

已冻结上传接口与 `MultipartFile` 参数，后续补充文件类型、大小校验、本地存储路径、下载响应及数据库记录。

### 7.4 Mock 数据与真实业务存在偏差风险

要求 Mock 路由、字段和状态枚举与接口文档一致，联调前执行一次字段对照检查。
