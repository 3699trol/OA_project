## 五、开发工作记录

### 5.1 数据模型与数据库初始化

- 调整 `JobApplication`、`Interview`、`InterviewEvaluation`、`InterviewQuestion`、`Job`、`Resume`、`SysUser` 和 `SysUserRole`，使字段类型、时间字段和逻辑删除配置与 `sql/init.sql` 保持一致。
- 新增 `MockInterview`、`MockInterviewRecord`、`ResumeEducation`、`ResumeExperience`、`AiAnalysis`、`Company`、`OperationLog` 和 `SysRolePermission` 实体。
- 修复初始化管理员 `admin` 的 Bcrypt 密码哈希，使其与文档中的 `admin123` 对应。
- 在初始化脚本中加入 `SET NAMES utf8mb4`，统一数据库连接字符编码。
- 增补概要设计文档，并将模块说明迁移到当前项目文档目录。

### 5.2 认证与系统管理接口

- 新增 `AuthService` 和 `AuthServiceImpl`，实现用户名查询、密码校验、JWT 生成、最后登录时间更新及用户注册。
- 调整登录响应为 `token + userInfo` 结构，包含用户 ID、用户名、角色、昵称和头像。
- 新增用户、角色、权限和操作日志 Service，实现分页查询、关键词筛选、状态过滤及时间范围过滤。
- 新增 `SysUserVO`，避免用户列表直接向前端返回密码字段。
- 管理端用户、角色、权限和操作日志页面开始接入真实响应，并补充失败提示。
- 退出登录、当前用户、刷新 Token、修改密码等认证接口仍为占位返回，不能视为认证模块全部完成。

### 5.3 职位管理与接口对齐

- 新增 `JobCreateRequest`、`JobUpdateRequest`、`JobService` 和 `JobServiceImpl`。
- 实现职位分页、关键词和状态筛选、详情、创建、更新、发布及下架业务逻辑。
- 前端 HR 职位列表、职位编辑、求职者首页和职位列表改用数字状态及统一的 `PageResult.records` 分页结构。
- 登录、注册、职位详情、投递记录和候选人列表等页面补充接口失败提示或字段兼容处理。
- 为 Maven 编译器开启 `-parameters`，并在职位、投递、面试、搜索、用户和日志 Controller 中显式声明 `@RequestParam(name = ...)`，降低运行时参数名解析失败风险。

### 5.4 前端体验与模拟面试

- 重构求职者简历编辑页，出生日期改为完整日期输入，并增加教育经历、工作经历的新增、编辑和删除弹窗。
- 完善经历时间段的解析、格式化、详情显示和空状态样式。
- 更新求职者、HR、面试官和管理员路由标题，移除主布局中的 Mock 模式提示。
- 在模拟面试页增加自定义 JD 编辑流程，可修改职位名称、薪资、城市、部门、职位描述和任职要求。
- 启动模拟面试时提交自定义的职位名称、描述和要求，为后续 AI 面试测试提供更灵活的输入。
- 前端全局强制关闭 Mock 模式，同步写入 `use_mock=false`，并清理无法通过真实 Spring Security 校验的演示 Token 和用户信息。

## 十、问题与风险

### 10.1 AI 服务依赖阻断全量构建

当前 Maven 镜像无法解析 `spring-ai-openai-spring-boot-starter:1.0.0-M1` 和 `spring-ai-core:1.0.0-M1`。进入 7 月 22 日 AI 攻坚前，应补充正确的 Spring Milestones 仓库或升级到可解析且与现有代码兼容的 Spring AI 版本。

### 10.2 前端已强制切换真实后端，但接口尚未全部实现

Mock 被全局关闭后，简历、投递、面试、搜索和文件页面会直接暴露后端占位接口问题。7 月 21 日应优先完成关键链路或提供明确、可控的环境开关，避免演示环境不可用。

### 10.3 注册角色字段存在权限风险

公开注册请求新增了 `userType`，服务端直接采用请求值。若不限制取值，客户端可能注册 HR、面试官甚至管理员账号。应由服务端固定普通注册为 `CANDIDATE`，管理角色只能由管理员分配。

### 10.4 接口状态缺少测试证据

API 文档已标记部分接口完成，但仓库没有接口自动化测试，且全量后端构建未通过。后续状态应至少以编译、数据库联调和关键请求响应三项证据为准。

### 10.5 配置样例缺失

`.env.example` 在上午实体补齐提交中新增，随后在后端接口提交中被删除。团队成员缺少统一环境变量模板，可能造成 MySQL、Nacos、Redis、ES 和 AI Key 配置差异。
