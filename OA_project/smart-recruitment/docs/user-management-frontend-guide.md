# 用户管理功能使用指南

## 功能说明

管理员可以在用户管理页面执行以下操作：
1. **查看用户列表** - 支持关键字搜索
2. **重置密码** - 将用户密码重置为默认密码 `123456`
3. **删除用户** - 逻辑删除用户（不可恢复）

## 操作步骤

### 1. 访问用户管理页面

- 使用管理员账号登录（用户名：`admin`，密码：`admin123`）
- 登录时选择角色：**管理员**
- 登录后会自动跳转到 `/admin/users` 用户管理页面

### 2. 重置用户密码

1. 在用户列表中找到需要重置密码的用户
2. 点击该用户行的 **"重置密码"** 按钮
3. 在弹出的确认对话框中点击 **"确定"**
4. 系统会显示成功提示：**"密码重置成功，新密码为：123456"**
5. 该用户可以使用新密码 `123456` 登录系统

### 3. 删除用户

1. 在用户列表中找到需要删除的用户
2. 点击该用户行的 **"删除"** 按钮
3. 在弹出的确认对话框中点击 **"确定删除"**
4. 系统会显示成功提示：**"用户删除成功"**
5. 用户列表会自动刷新，被删除的用户不再显示

### 4. 搜索用户

- 在页面顶部的搜索框中输入关键字
- 支持按用户名、姓名、邮箱、手机号搜索
- 搜索结果会实时更新

## 功能特性

### 确认对话框
- **重置密码**：黄色警告图标，提示将密码重置为 123456
- **删除用户**：红色错误图标，提示操作不可恢复

### 成功提示
- 操作成功后会在页面顶部显示绿色成功提示
- 提示会在 3 秒后自动消失

### 错误处理
- 如果操作失败，会显示红色错误提示
- 错误信息会说明失败原因（如：用户不存在）

## 注意事项

1. **重置密码**
   - 重置后的密码统一为 `123456`
   - 建议通知用户及时修改密码
   - 不建议重置管理员账号密码

2. **删除用户**
   - 删除操作是逻辑删除，数据库中 `deleted` 字段标记为 1
   - 删除后用户无法登录系统
   - 不建议删除管理员账号，避免无法管理系统

3. **权限控制**
   - 只有管理员角色可以访问用户管理页面
   - 其他角色访问会被拒绝

## Mock 模式测试

如果前端独立运行（Mock 模式），功能同样可用：
- 重置密码会在控制台输出日志
- 删除用户会在控制台输出日志
- 但不会真正修改数据库

## 真实后端测试

连接真实后端时：
1. 确保后端服务已启动（端口 8080）
2. 确保数据库连接正常
3. 操作会真实修改数据库数据
4. 可以在数据库中验证结果

## API 接口

- **重置密码**: `PUT /api/system/user/{id}/reset-password`
- **删除用户**: `DELETE /api/system/user/{id}`
- **用户列表**: `GET /api/system/user/list?keyword=xxx`
- **用户详情**: `GET /api/system/user/{id}`

## 常见问题

**Q: 点击按钮没有反应？**
A: 检查浏览器控制台是否有错误，确认后端服务是否正常运行。

**Q: 提示"用户不存在"？**
A: 该用户可能已被删除，刷新页面重新加载用户列表。

**Q: 重置密码后用户无法登录？**
A: 确认使用的是新密码 `123456`，并且选择了正确的角色。

**Q: 删除后用户还能看到？**
A: 刷新页面，用户列表会重新加载。
