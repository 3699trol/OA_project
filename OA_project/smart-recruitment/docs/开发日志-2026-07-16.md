## 四、开发工作记录

### 4.1 公共后端能力

- 建立 `common-core`，提供 `Result`、分页结果、错误码和业务异常。
- 建立 `common-web`，提供全局异常处理、请求 ID、用户上下文拦截器和操作日志注解。
- 建立 `common-security`，提供 `SecurityConfig`、JWT 工具、认证过滤器和登录用户模型。
- 建立 `common-redis`，封装 Redis 配置及常用读写操作。
- 统一 Java 编译版本为 JDK 21，并确认 Maven 多模块依赖关系。

### 4.2 认证与用户模块

- 创建 `SysUser`、`SysRole`、`SysPermission`、`SysUserRole` 实体及 Mapper。
- 创建登录请求、注册请求和登录响应对象，避免直接向前端暴露数据库实体。
- 定义注册、登录、退出、当前用户、刷新 Token 和修改密码接口路径。
- 明确密码使用 BCrypt 存储、Token 使用 Bearer 方式传递的安全约定。
- 当前认证 Controller 已完成接口骨架，`UserDetailsService`、数据库校验及完整注册业务仍需继续实现。

### 4.3 前端认证流程

- 使用 Axios 创建统一请求实例，设置 `/api` 基础路径和超时时间。
- 请求拦截器预留 JWT Token 注入，响应拦截器统一处理业务错误和未授权跳转。
- 使用 Pinia 保存 Token 与用户信息，并同步到 LocalStorage。
- 完成登录、注册页面的表单结构与角色入口。
- 配置求职者、HR、面试官、管理员路由及基础访问守卫。

### 4.4 启动与配置优化

- 新增 `local` 轻量启动配置，使核心服务可在不连接 MySQL、Redis 和 ES 的情况下验证项目结构。
- 保留 `dev` 完整联调模式，用于后续数据库及中间件接入。
- 补充开发环境要求、数据库字段说明和前后端启动步骤。
- 统一核心服务端口 `8080`、AI 服务端口 `8081`、前端端口 `3000`。

### 4.5 测试记录

- 检查登录、注册表单必填项和密码长度规则。
- 检查无 Token 访问受保护路由时的跳转行为。
- 检查 Token 和用户信息的本地持久化结构。
- 后端真实登录仍返回占位结果，数据库用户校验和 Token 签发未达到完整联调标准。

## 七、问题与处理记录

### 7.1 认证接口只有结构，尚无完整业务层

为保证其他成员可以并行开发，先冻结接口路径和数据结构；后续补充 AuthService、UserDetailsService、BCrypt 校验及角色查询。

### 7.2 完整中间件影响首次启动

增加 `local` profile 排除数据源和中间件自动配置，先验证工程可启动；需要真实业务数据时切换到 `dev` profile。

### 7.3 前后端角色字段需统一

统一使用 `ADMIN`、`HR`、`INTERVIEWER`、`CANDIDATE`，避免页面路由、Token 权限和数据库角色编码不一致。
