# 🌟 Smart Recruit 前端现代界面与智能 Mock 系统使用指南

企业级智慧招聘与人才管理系统（Smart Recruit）的前端模块已完成重构。此版本融合了当下最前沿的 **莫兰迪极简设计规范** 与 **零依赖无感 Mock 拦截系统**。您无需启动后端 Java 微服务或配置本地环境，即可一键独立运行并完整体验招聘 OA 系统的所有核心交互和业务流程。

---

## 🎨 1. 极致打磨的 UI/UX 设计质感

为了消除传统 OA 系统拥挤、冰冷的感觉，此版本对界面进行了精细雕琢，严格遵守以下美学规范：
*   **完美平衡的黄金间距 (Breathing Room)**：
    *   页面采用 `24px` / `32px` 黄金分割的外边距，表单及卡片内容使用 `16px` / `20px` 内边距。组件间保持温和的间距，**不拥挤亦不分散**。
*   **克制协调的莫兰迪调色板**：
    *   **侧边栏 (Aside)**：摒弃了原本老旧的沉重泥土深棕色，升级为高级曜墨黑 (`#1a1d21`)。文字使用优雅的雾白 (`#a3aed0`)，高亮时采用温润的晨曦暖橙微胶囊选中效果，散发沉稳而智能的专业气场。
    *   **主色调 (Primary)**：微调为更具质感的晨曦暖橙赤金色 (`#e76f51`)，用于核心按钮和操作，极具现代品牌设计质感，不晃眼、不流俗。
    *   **主背景色 (Background)**：采用淡冷雾灰 (`#f8f9fc`) 作为整站底色，使得纯白色卡片具有极致的浮雕感与高阶悬浮层次。
*   **智能轻量微卡片与图表**：
    *   卡片采用微圆角 (`12px`) 并除去厚重的实体边框，替换为极其温柔的微风轻抚雾气阴影 (`box-shadow: 0 4px 18px -4px rgba(0,0,0,0.04)`)。
    *   看板图表抛弃了冗长笨重的第三方 ECharts 组件加载，采用高雅的 **纯原生 SVG 矢量渐变折线面积图** 与 **动态环形进度**，首屏秒级展现，美感十足。

## 🚀 2. 极速本地独立运行步骤

前端基于 **Vue 3 (Vite) + Element Plus + Pinia** 构建。请按照以下命令启动：

```bash
# 1. 导航到前端 recruitment-web 目录
cd recruitment-web

# 2. 安装项目依赖（已锁定最优包版本）
npm install

# 3. 一键启动本地 Vite 开发服务器
npm run dev
```

*启动成功后，浏览器通常会自动打开 [http://localhost:3000](http://localhost:3000) 即可体验。*

## 🛡️ 3. 智能无感 Mock 拦截系统

为了确保前后端开发的彻底解耦与流畅测试，我们在请求底层植入了无感 Mock。它直接在 `axios` 拦截器中根据请求的 `URL` 与 `Method` 自动匹配本地模拟路由。

### 🌟 核心特长
1.  **零依赖、零配置**：不需要安装容易导致 Windows 依赖报错的外部 native packages。
2.  **高动态流转**：内存中保留状态，支持在招职位的实时**发布、下架、过滤搜索、Tabs 分流**等仿真联动。
3.  **真实网络延迟呼吸感**：默认对接口返回注入 `350ms` 的真实网络延时，能够完美展示前端骨架屏、Loading 加载动画等高仿真细节。
4.  **无缝过渡**：Mock 模拟接口路径、入参、出参和真实后端 API 规范 100% 一致。当后端就绪时，**不需要改动前端任何业务代码**，仅需关掉 Mock 开关即可瞬间无缝对接到真实服务器。

## 💡 4. 前端 Mock 调试与使用方法

为了给求职、面试和演示人员提供最丝滑的操作闭环，我们设计了以下机制：

### 🔑 4.1 四大职场角色“一键快捷登录”
在登录界面中，我们提供了 **快速体验区**（点击对应的彩色标签，即可瞬间完成账号填充并自动登录）：
1.  **求职者 🧑‍💼 (张三)**：拥有完整的求职首页、公开职位浏览投递、我的智能简历、投递进度追踪、**AI 仿真模拟面试对练**等全套求职闭环。
2.  **HR经理 👩‍💼 (李经理)**：可体验全链路招聘监控大屏、发布与管理职位、人才库简历甄选、安排面试、**AI 岗位人岗匹配度深度分析**等核心 OA 功能。
3.  **技术面试官 👨‍🏫 (王工)**：拥有待面试任务面板、**AI 面试智能出题器**、评审面试历史。
4.  **超级管理员 🛡️ (Admin)**：全局用户管理、角色权限控制、系统审计日志和 AI 参数配置面板。

### 🔄 4.2 Mock 模式与 真实后端 Prod 模式的一键切换
我们极度体贴地在页面的 **Header 右侧**（头像左边）提供了一个一键切换开关：
*   **一键切换**：点击顶栏亮绿色的 `Mock 模式` 标签，系统会自动将 `localStorage` 中的 `use_mock` 设为 `false`，并自动刷新页面，此时页面将彻底关闭 Mock 拦截，**直连您本地 `8080` 端口的 Java 服务**（通过 `vite.config.js` 的代理直接转发）。
*   **手动复原**：再次点击，或者在浏览器 F12 控制台运行：
    ```javascript
    localStorage.setItem('use_mock', 'true') // 启用本地 Mock 独立运行 (默认)
    localStorage.setItem('use_mock', 'false') // 关闭 Mock 直连后端
    ```

## 📦 5. 覆盖全业务流的 Mock 接口清单

拦截系统已完美注册并支持了以下核心业务接口，您在页面里执行的动作将实时联动：

| 模块 | 接口路由 | 匹配方式 | 模拟业务逻辑 / 联动流转 |
| :--- | :--- | :--- | :--- |
| **安全认证** | `POST /api/auth/login` | 精确匹配 | 校验凭证并返回包含 CANDIDATE/HR/INTERVIEWER/ADMIN 的 token |
| | `GET /api/auth/current-user` | 模糊匹配 | 根据 Request 头的 Bearer Token 解析对应身份，实现保持登录 |
| **数据监控** | `GET /api/dashboard/stats` | 精确匹配 | 获取看板 在招职位、投递数、面试数及最新 5 条简历等汇总指标 |
| **职位管理** | `GET /api/job/list` | 正则匹配 | 支持根据 **职位标题模糊搜索** 与 **类别下拉筛选**，支持 Tabs 状态分流 |
| | `POST /api/job/:id/publish` | 路径正则 | 联动修改该职位状态为 `PUBLISHED`，可即时在前台职位大厅搜到 |
| | `POST /api/job/:id/unpublish` | 路径正则 | 联动修改职位状态为 `UNPUBLISHED`，从前台搜索列表中隐藏 |
| **投递处理** | `POST /api/application` | 精确匹配 | 触发投递，求职者投递后该职位会在 HR 的监控面板中即时出现 |
| | `PUT /api/application/:id/status` | 路径正则 | 联动更新投递流程，如待筛选 -> 面试中 -> 录用 / 淘汰 |
| **面试评审** | `POST /api/interview` | 精确匹配 | 支持 HR 快速为通过初选的候选人预约线上会议和线下场次 |
| **人工智能** | `POST /api/ai/match-analysis` | 精确匹配 | **AI 人岗匹配评分**，计算并输出大模型简历契合度与决策结论 |
| | `POST /api/ai/mock-interview/start` | 精确匹配 | **AI 模拟面试开启**，初始化大语言模型并输出开场白 |
| | `POST /api/ai/mock-interview/chat` | 精确匹配 | **AI 面试实时对练**，支持上下文对话并推荐启发式提问 |
| | `POST /api/ai/mock-interview/submit` | 精确匹配 | **AI 智能报告生成**，一键提交对练，生成多维度打分和专业建议 |
| **管理后台** | `GET /api/admin/log/list` | 精确匹配 | 展示管理员、HR 实时对系统参数修改的详细后台审计日志 |

此套 Mock 系统提供了一个完美的开发底座。希望这套充满设计细节、克制高档的前端体验，能够完美契合您的招聘项目演示和产品评审！
