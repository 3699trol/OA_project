七、建议的生成顺序

第一批直接生成基础CRUD：

 

sys_user

sys_role

job

resume

job_application

interview

interview_question

interview_evaluation

第二批手动实现核心业务：

 

登录与JWT鉴权

数据权限

职位投递状态流转

HR分配面试官

正式面试题生成

面试评价

AI人岗匹配

模拟面试

Elasticsearch同步

RedisStack向量检索

最终最小可运行框架为：

 

前端：Vue 3 + Element Plus + Pinia + Vue Router

核心服务：Spring Boot + Security + MyBatis + Redis + ES

AI服务：Spring Boot + Spring AI + RedisStack

服务治理：Nacos + OpenFeign

数据库：MySQL

这个划分既能生成基础代码，又将HR、面试官和求职者的AI功能明确隔离。
